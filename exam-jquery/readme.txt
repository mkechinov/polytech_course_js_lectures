ФОРМАТ ЭКЗАМЕНА

1. Сделать приложение на основе техзадания.
2. Устно защитить его на экзамене.



КРАТКОЕ ОПИСАНИЕ ПРИЛОЖЕНИЯ

Электронная ведомость для проведения экзамена. Экзаменатор имеет возможность создать запись, связующую студента, предмет, оценку и дату.



ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

- приложение сделано с использованием jQuery и jQueryUI.
- для выбора даты используется Datepicker.
- для формы добавления или редактирования записи используется Dialog.
- для выбора студента используется Autocomplete.
- оценка текстовая, варианты: неявка, неудовлетворительно, удовлетворительно, хорошо, отлично.
- таблица с оценками - простая плоская таблица как в Excel без группировок и усложнений.



ПОДГОТОВИТЕЛЬНЫЕ РАБОТЫ

- подготовить массив фамилий всех студенов вашей группы.
- подготовить массив всех ваших предметов.



ОСНОВНЫЕ СЦЕНАРИИ ИСПОЛЬЗОВАНИЯ ПРИЛОЖЕНИЯ


Сценарий 1. Экзаменатор может изучить список оценок.

Последовательность шагов:
1. Экзаменатор открывает приложение.
2. Экзаменатор видит таблицу со списком оценок, состояющую из колонок: фамилия, предмет, дата, оценка, иконка редактирования записи.
3. Экзаменатор видит кнопку добавления записи.

Варианты развития событий:
1. Оценок еще нет, поэтому экзаменатор видит пустую таблицу.


Сценарий 2. Экзаменатор может добавить оценку.

Последовательность шагов:
1. Экзаменатор нажимает на кнопку "Добавить запись". Открывается диалоговое окно с формой добавления оценки.
2. Экзаменатор заполняет поле с фамилией студента. Autocomplete подсказывает существующие фамилии.
3. Экзаменатор выбирает предмет из выпадающего списка.
4. Экзаменатор выбирает дату с помощью Datepicker. По-умолчанию выбрана текущая дата.
5. Экзаменатор выбирает оценку.
6. Экзаменатор нажимает кнопку "Сохранить".
7. Dialog закрывается, таблица перерисовывается и в ней есть только что добавленная запись.

Варианты развития событий:
1. Не заполнено одно из полей. При попытке сохранить на экране выводится сообщение о том, какое поле не заполнено.


Сценарий 3. Экзаменатор может отредактировать оценку.

Последовательность шагов:
1. Экзаменатор нажимает на иконку редактирования напротив записи.
2. Открывается диалоговое окно с формой, как в создании, но без выбора фамилии.
3. Экзаменатор заполняет дату, оценку и предмет.
4. Экзаменатор нажимает кнопку "Сохранить".
5. Диалог закрывается. Экзаменатор видит измененный список записей.




БОНУСНЫЕ ЗАДАЧИ

При решении бонусной задачи получаете +0.5 балла за каждую.

- перед добавлением записи выполняется проверка: есть ли в ведомости связка студент-предмент. Если есть, то оценка заменяется, а не добавляется.
- после редактирования записи выполняется проверка: если в списке уже есть связка студент-предмет и это не текущая редактируемая связка, то старая запись перезаписыватся новой записью.
- выбор оценки сделать с помощью Slider.



ПОРЯДОК ОЦЕНКИ ИСПОЛНЕНИЯ

1. Список, создание и редактирование – 5 баллов.
2. Не использовать autocomplete - минус 1 балл.
3. Не использовать datepicker - минус 1 балл.
4. Оценка не текстовая – минус 1 балл.
5. Создание или редактирование не в Dialog – минус 1 балл.
